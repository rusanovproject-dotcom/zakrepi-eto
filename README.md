# «Закрепи это» — станок для скиллов

Ты поработал с Claude, получил хороший результат — и не хочешь объяснять то же самое заново в следующий раз.

Говоришь: **«закрепи это»**. Claude смотрит, что вы только что делали, вытаскивает оттуда порядок действий и твои правки, задаёт пару вопросов — и собирает скилл. Скилл — это файл-памятка, который в следующий раз сработает сам, без напоминаний.

Ставится один раз. Дальше ты просто разговариваешь.

---

## Установка

### Шаг 1. Скачай папку

Скачай `zakrepi-eto`. Если скачался архив (`.zip`) — распакуй его двойным щелчком. Должна получиться **папка** с именем `zakrepi-eto`, а внутри неё — файл `SKILL.md`.

Обычно всё это оказывается в «Загрузках». Запоминать, где именно, не нужно — Claude найдёт сам.

### Шаг 2. Открой Claude Code

Claude Code — это Claude, который живёт у тебя на компьютере и умеет работать с файлами. Не сайт claude.ai, а именно та программа, которую ты ставил на курсе. Открой её и дождись строки ввода.

### Шаг 3. Скажи ему одну фразу

Скопируй это целиком и вставь в Claude Code как обычное сообщение. Ничего не меняй и ничего не подставляй — он разберётся сам:

```
Найди у меня на компьютере папку zakrepi-eto (скорее всего она в Загрузках)
и установи её как скилл Claude Code: она должна лежать в ~/.claude/skills/zakrepi-eto,
а имя папки — остаться ровно zakrepi-eto. Потом скажи мне, что получилось.
```

Он найдёт папку, положит куда надо и отчитается. Терминал открывать не нужно — это его работа, не твоя.

### Шаг 4. Перезапусти Claude Code

Новую папку со скиллами он замечает только при запуске. Поэтому:

1. Закрой окно Claude Code — обычным крестиком, как любую программу. Ничего не сломается: всё уже сохранено на диск.
2. Открой заново.

### Шаг 5. Проверь, что он видит скилл

В строке ввода набери:

```
/skills
```

В списке должен появиться **zakrepi-eto**.

Есть в списке — всё, установка закончена.

Нет в списке — скопируй и вставь эту фразу:

```
Скилла zakrepi-eto нет в списке /skills. Найди, куда легла папка zakrepi-eto,
проверь, что она лежит в ~/.claude/skills/ и что имя папки не изменилось,
и почини. Частая причина: файлы легли в ~/.agents/skills/ вместо ~/.claude/skills/.
```

Он поймёт, даже если это уже новый разговор и он не помнит, что было раньше: в этой фразе есть всё, что ему нужно.

## Как пользоваться

Поработал с Claude. Получилось хорошо. Говоришь любую из этих фраз:

- «закрепи это»
- «сделай из этого скилл»
- «чтобы больше не объяснять тебе это каждый раз»
- «в следующий раз чтобы сразу так»

Дальше он сам:

1. **Посмотрит, что вы делали**, и вытащит порядок действий — вместе с **твоими правками**. Каждое твоё «нет, не так» превращается в правило. Это и есть самое ценное в скилле: без правок он был бы пустой.
2. **Честно скажет, если это не тянет на скилл.** Разовая задача скиллом не становится — он так и ответит, а не станет плодить мусор. Почему — объяснено ниже.
3. **Задаст не больше трёх вопросов.** Главный: «какими словами ты будешь просить это в следующий раз?»
4. **Соберёт файл и проверит его** на живой задаче — сравнит, как выходит со скиллом и без него.
5. **Скажет, где файл лежит и какой фразой его звать.**

Не сработал в следующий раз сам? Скажи: **«скилл не сработал»**. Он починит — дело почти всегда в описании, и он знает, как его переписать.

---

## Почему он иногда отказывается делать скилл

У Claude ограниченное место под список скиллов. Когда скиллов становится слишком много, он начинает обрезать их описания — и ломается срабатывание тех, что прекрасно работали раньше. Один лишний скилл портит жизнь всем остальным.

Поэтому станок сначала честно взвешивает: это правда повторяющаяся работа или разовая задача? Отказ — это защита твоих остальных скиллов, а не отговорка.
